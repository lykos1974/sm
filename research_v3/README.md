# Research Operating System v1 (ROS-1)

ROS-1 is minimal registry scaffolding for preserving research knowledge. It is documentation and registry structure only.

- Detectors remain plugins outside ROS-1.
- No trading logic lives here.
- Registries preserve research knowledge; they do not execute it.
- Every research record must trace through: Observation → Belief → Experiment → Evidence → Decision → Knowledge.
- No promotion is allowed without validation evidence.
- Registry creation must not modify the protected Pole baseline.
- The protected baseline remains LONG only, `POST_BREAKOUT_PULLBACK`, `HEALTHY`, `active_leg_boxes: 2`, non-extended only, all symbols in scope, and no early breakeven logic.

ROS-1 does not implement databases, backtests, live trading, validation, detector development, or strategy development.
